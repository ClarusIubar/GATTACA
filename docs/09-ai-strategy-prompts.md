# 09. AI 활용 전략과 프롬프트 자산

## AI 역할 정의
- 문서 정리자
- 코드 초안 작성자
- UI 구조화 보조자
- 테스트 케이스 초안 보조자

## 공통 입력 규칙
- 프로젝트 비전:
  - 추억열차는 카카오톡 단체방에서 확정된 일정과 추억 기록을 보관하는 메모리얼 웹앱이다.
- 기술 스택:
  - React, TypeScript, Vite, Supabase, GitHub Pages
- 산출물 형식 기준:
  - 한국어 우선
  - TypeScript 기준
  - 문서와 코드 동시 갱신
- AI 금지 사항:
  - 민감정보 하드코딩
  - 이해 없는 복사 붙여넣기 유도
  - 카카오 API 가능 범위 과장

## 잘 작동한 프롬프트 패턴
- “문서와 구현을 함께 갱신하고, 권한 모델을 먼저 설명하라.”
- “GitHub Pages 정적 제약을 고려해 가능한 구현만 제안하라.”
- “README, docs, wiki 원고까지 하나의 일관된 체계로 작성하라.”

## 실패 프롬프트 예시
- “카카오톡 단체방 멤버를 API로 자동 검증하라.”
  - 이유: 공개 API 제약으로 실제 구현 불가

## 결과 검토 기준
- 사실성: 카카오, Supabase, Pages 제약을 정확히 반영했는가
- 맥락 적합성: 추억열차 콘셉트와 맞는가
- 과잉 복잡성: 소규모 운영에 비해 과도하지 않은가
- 유지보수 리스크: 문서와 코드가 쉽게 깨지지 않는가
