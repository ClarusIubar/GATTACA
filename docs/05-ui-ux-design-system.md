# 05. UI/UX 디자인 시스템

## 디자인 원칙
- 기억을 불러오는 따뜻한 질감
- 일정 관리보다 장면 회상에 집중
- 복잡한 기능보다 자연스러운 기록 흐름 우선

## 시각 시스템
- 주 색상: `#9e4f2a`
- 보조 색상: `#c89537`
- 배경 톤: `#f9f4eb`, `#efe2ce`
- 경고/오류: `#9b3b33`
- 본문 폰트: `Segoe UI`, `Apple SD Gothic Neo`, sans-serif
- 강조 폰트: `Georgia`, `Times New Roman`, serif
- 간격 단위: 8px 기반

## 핵심 컴포넌트
- 이벤트 카드: 시간, 장소, 요약, 이동 액션
- 메모리 카드: 작성자, 시각, 사진, 캡션, 코멘트 영역
- 상태 배너: 데모 모드, 승인 대기, 오류 메시지
- 관리자 카드: 승인 대기 목록, 운영 대상 이벤트
- 폼: 일정 등록, 메모리 등록, 코멘트 등록/수정

## 핵심 사용자 흐름
- 홈 진입 → 이벤트 목록 → 이벤트 상세 → 메모리/코멘트 열람
- 승인 사용자 로그인 → 일정 등록 → 이벤트 상세에서 추억 누적
- 운영자 로그인 → 승인 대기 확인 → 승인/반려 처리

## 접근성 및 반응형
- 모바일 우선 스택형 레이아웃 지원
- 모든 입력 필드에 라벨 제공
- 버튼과 링크 구분 유지
- 승인 불가 상태에서 disabled 처리와 안내 문구 동시 제공
