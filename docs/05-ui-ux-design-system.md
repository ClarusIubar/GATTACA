# 05. UI/UX 디자인 시스템

## 디자인 원칙

- “추억열차” 콘셉트를 화면 언어의 중심으로 둔다.
- 복잡한 일정 관리보다, 지나간 약속과 장면을 다시 꺼내 보는 감각을 우선한다.
- 카카오톡 단체방은 결정의 장소, 추억열차는 기록의 장소라는 역할 분리를 분명히 보여준다.
- 권한 상태는 숨기지 않고 배너, 버튼 disabled, 운영 안내 문구로 명확히 표시한다.

## 시각 시스템

- 주 색상: `#9e4f2a`
- 보조 색상: `#c89537`
- 배경색: `#f9f4eb`, `#efe2ce`
- 강조/오류: `#9b3b33`
- 본문 폰트: `Segoe UI`, `Apple SD Gothic Neo`, sans-serif
- 강조 폰트: `Georgia`, `Times New Roman`, serif
- 간격 단위: 8px 기반

## 핵심 화면

- 홈: 운행 상태 보드, 기록 노선도, 최근 정거장, 운영 원칙.
- 목록: 정거장형 타임라인 카드. 월간 캘린더는 사용하지 않는다.
- 상세: 이벤트 요약, 무엇을/어떻게, 메모리 갤러리, 코멘트 흐름, 권한별 수정/삭제.
- 등록: 확정된 일정을 정거장으로 등록하는 집중 폼.
- 소개: 콘셉트, 운영 원칙, v1 경계를 설명한다.
- 운영실: 승인 대기자와 삭제 권한의 관리 화면.

## 컴포넌트 규칙

- 정거장 카드: 날짜, 장소, 등록자, 확정 요약, 상세 CTA를 포함한다.
- 출발 보드: 접근 가능 여부, 백엔드 연결, Kakao 상태, 기록 수를 보여준다.
- 메모리 카드: 사진, 작성자, 기록 시각, 캡션, 코멘트 목록을 포함한다.
- 폼: 모든 입력 필드에 명시적 label을 제공한다.
- 상태 배너: 데모 모드, 승인 대기, Kakao 미설정, 로딩, 오류를 구분한다.
- 위험 버튼: 삭제 동작에만 사용한다.

## 접근성과 반응형 기준

- 모든 입력은 label 또는 aria-label로 접근 가능해야 한다.
- 링크와 버튼의 역할을 혼동하지 않는다. 이동은 링크, 상태 변경은 버튼을 사용한다.
- 승인 전 작성 기능은 disabled 처리와 안내 문구를 동시에 제공한다.
- 모바일에서는 two-column 레이아웃을 단일 컬럼으로 접는다.
- 이미지 실패 시 fallback 이미지를 보여준다.

## UX 검증 기준

- 홈 hero에서 “단톡방의 약속을 오래 남는 정거장으로.” 문구가 보여야 한다.
- 목록 CTA는 “새 정거장 등록”, 상세 진입은 “정거장 상세 보기”로 고정한다.
- 등록 폼은 “이벤트 제목”, “언제”, “어디서”, “무엇을 할지”, “어떻게 진행할지”, “확정 요약”을 포함한다.
- 상세 메모리 폼은 “메모리 캡션”, “기록 시각”, “또는 사진 URL”, “메모리 남기기”를 포함한다.
- 코멘트 입력은 “코멘트 남기기”, 제출은 “코멘트 추가”로 검증한다.
