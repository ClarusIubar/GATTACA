# 02. 팀 역할과 운영 규칙

## 역할과 책임

| 역할 | 책임 범위 | 결정권 / 검토권 |
| --- | --- | --- |
| 운영자 | 제품 범위, 승인 정책, 삭제 권한, 배포 승인 | 최종 결정권 |
| 승인 사용자 | 이벤트/메모리/코멘트 작성 및 수정 | 작성 책임 |
| AI 보조 | 문서 초안, UI/코드 초안, 테스트 초안 | 제안 및 보조 |

## 소통 규칙
- 일정과 투표 결과의 최초 소스: 카카오톡 단체방
- 제품 내 기록 기준 문서: 저장소 `docs/`
- 개발 진행 공유: GitHub README, Wiki 개발기록, 커밋 로그

## 작업 운영 규칙
- 요구사항 변경 시 PRD와 구현 계획을 먼저 수정한다.
- 지연/실패 발생 시 문서 10 테스트 이슈와 문서 12 회고에 기록한다.
- 권한 정책 변경은 문서 00, 04, 06, 11 동시 점검이 필요하다.

## 리뷰 문화
- 피드백은 사람보다 산출물에 집중한다.
- 권한, 보안, 삭제 정책, 문서-코드 불일치를 우선 검토한다.
