# 제품 요구사항 정의서 (Product Requirement Document - PRD)

이 문서는 `추억열차 (GATTACA)`의 현재 제품 요구사항과 구현 완료 경계를 정의합니다. 기준은 "카카오 API만 붙이면 되는 수준"까지의 실사용 MVP 재구축입니다.

---

## 1. 제품 비전

추억열차는 카카오톡 단체방에서 흩어지는 **모임의 확정 결과와 그날의 추억**을 한곳에 남기는 메모리얼 타임라인 서비스입니다.

- **기본 철학**: 단톡방에서는 논의와 결정이 일어나고, 추억열차는 확정된 일정과 회고만 기록한다.
- **운영 원칙**: 단톡방 구성원만 쓰기 권한을 가져야 하며, 삭제는 운영자만 가능하다.
- **기술 방향**: 프론트는 Cloudflare Pages, 백엔드는 Cloudflare Workers, 데이터는 D1/R2/KV를 사용한다.

---

## 2. 사용자 역할

1. **게스트**
   - 공개 페이지와 기록을 읽을 수 있다.
   - 쓰기 불가

2. **승인 대기 사용자**
   - Kakao 로그인 직후 생성
   - 읽기 가능
   - 이벤트/메모리/코멘트 작성 불가

3. **승인 사용자**
   - 운영자 승인 완료
   - 이벤트 작성/수정 가능
   - 메모리 작성/수정 가능
   - 코멘트 작성/수정 가능
   - 삭제 불가

4. **운영자**
   - 승인/반려 가능
   - 이벤트/메모리/코멘트 삭제 가능

---

## 3. 핵심 기능 요구사항

### REQ-01. 확정 일정 기록

- 일정은 "언제, 어디서, 무엇을, 어떻게"와 결정 요약을 포함해야 한다.
- 이벤트 목록은 타임라인 성격을 유지해야 한다.
- 캘린더형 월간 UI는 필수 요구사항이 아니다.

### REQ-02. 이벤트별 메모리 기록

- 각 이벤트 상세에서 사진과 메모를 남길 수 있어야 한다.
- 사진은 실제 저장소에 올라가야 한다.
- 메모리에는 작성자와 기록 시점이 연결되어야 한다.

### REQ-03. 코멘트/방명록

- 이벤트 메모리에 코멘트를 남길 수 있어야 한다.
- 승인 사용자만 작성 가능해야 한다.

### REQ-04. Kakao 로그인과 승인 기반 접근 통제

- Kakao OAuth로 로그인해야 한다.
- 첫 로그인 사용자는 `pending`으로 생성되어야 한다.
- 승인 전에는 쓰기 기능이 차단되어야 한다.
- 운영자만 승인/반려 가능해야 한다.

### REQ-05. 운영자 삭제 권한

- 이벤트 삭제는 운영자만 가능
- 메모리 삭제는 운영자만 가능
- 코멘트 삭제는 운영자만 가능

### REQ-06. Kakao relay 경계

- 브라우저가 Kakao API를 직접 호출하면 안 된다.
- Worker가 세션의 Kakao access token을 사용해 relay 해야 한다.
- 이벤트 저장 성공 후 relay 실패는 부분 성공으로 표시해야 한다.

---

## 4. 비기능 요구사항

### 4.1 모바일 카카오 인앱 브라우저 호환

- 카카오톡 인앱 브라우저에서 레이아웃이 깨지지 않아야 한다.
- 공개 읽기와 로그인 유도 동선이 모바일에서 바로 작동해야 한다.

### 4.2 배포 및 성능

- Cloudflare Pages/Workers 기반으로 배포 가능해야 한다.
- production에서 demo fallback이 실사용처럼 보이면 안 된다.

### 4.3 보안

- Kakao secret은 프론트에 노출되면 안 된다.
- Worker secret으로만 관리해야 한다.
- 권한은 프론트가 아니라 Worker가 강제해야 한다.

### 4.4 운영 명확성

- live에서 Kakao secret이 없으면 UI가 그 상태를 숨기지 않아야 한다.
- `/api/runtime-status`로 현재 배포 상태를 readback할 수 있어야 한다.

---

## 5. 현재 구현 완료 범위

현재 저장소는 아래 범위까지 구현되어 있다.

- Cloudflare Worker 백엔드
- D1 CRUD
- KV 세션 저장/복원
- R2 업로드
- 서버 측 승인/운영자 권한 강제
- Kakao OAuth callback
- Kakao relay endpoint
- production demo fallback 차단
- live runtime readback 문서화

---

## 6. 현재 미완료 범위

아직 남은 것은 저장소 내부 구현이 아니라 live 외부 secret과 실계정 검증이다.

- `KAKAO_REST_API_KEY`
- `KAKAO_CLIENT_SECRET`
- 실제 Kakao login redirect/callback/session restore 확인
- 실제 Kakao relay 수신 확인

즉 현재 완료 경계는 다음과 같다.

- **도달한 상태**: 카카오 API만 붙이면 되는 수준
- **아직 미증명인 상태**: live Kakao account 기준 end-to-end 완료
