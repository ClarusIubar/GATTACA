# 10. 테스트와 QA 전략

## 테스트 범위
- 단위 테스트:
  - 홈 화면 핵심 문구 렌더링
  - 환경변수 미설정 시 데모 배너 노출
- 통합 테스트:
  - 이벤트 상세에서 메모리/코멘트 흐름
  - 권한 상태별 버튼 활성/비활성
- 수동 QA:
  - 데모 권한 전환
  - Supabase 로그인/승인 흐름
  - GitHub Pages 라우팅 새로고침

## 테스트 우선순위
1. 비로그인/승인대기/승인회원/운영자 권한 흐름
2. 이벤트 등록 → 상세 진입 → 메모리 추가 → 코멘트 추가
3. 운영자 승인/삭제 흐름

## 품질 기준
- 린트 오류 0건
- 타입 오류 0건
- 핵심 홈 렌더링 테스트 통과
- 프로덕션 빌드 통과

## 릴리스 전 필수 검증
- `.env.local` 없이 데모 모드 실행 확인
- Supabase 연결 시 로그인 버튼 동작 확인
- 관리자 승인 페이지 접근 통제 확인
- 404 새로고침 대응 확인

## 현재 기술 부채
- Supabase 실환경 E2E는 아직 자동화되지 않음
- 이미지 업로드 진행률/실패 재시도 UX 미구현
