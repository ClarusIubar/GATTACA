# 01. 프로젝트 비전과 성공 지표

## 존재 이유
- 카카오톡 단체방에서 함께 정한 일정은 대화 흐름 속에서 빠르게 묻힌다.
- 추억열차는 합의된 일정과 그날 남긴 사진/코멘트를 한 자리에 보관해, 단체의 기억을 반복해서 꺼내볼 수 있는 페이지를 만든다.

## 만들고 싶은 변화
- 일정 결정과 추억 기록이 분리되지 않도록 한다.
- 단체방 구성원만 참여하는 소규모 메모리얼 공간을 제공한다.
- 채팅 로그 대신 정리된 이벤트 카드와 메모리 카드로 기록 품질을 높인다.

## 지금 시작해야 하는 이유
- 단체방 기반 소규모 커뮤니티는 기록 자산이 쉽게 흩어진다.
- GitHub Pages와 Supabase만으로도 작은 운영 비용으로 시작 가능하다.

## 핵심 가치
- 기억의 지속성
- 단순한 운영
- 승인 기반 신뢰

## 성공 기준
- 정성:
  - 운영자가 일정과 추억을 다시 찾기 쉬워야 한다.
  - 구성원이 메모리와 코멘트를 부담 없이 남길 수 있어야 한다.
- KPI:
  - 첫 1개월 내 이벤트 3건 이상 기록
  - 이벤트당 메모리 2건 이상
  - 승인 완료 사용자의 재방문 기록 생성률 50% 이상
- 실패 신호:
  - 승인 흐름이 복잡해 가입 후 이탈이 높음
  - 이벤트 등록은 되지만 추억 기록이 이어지지 않음

## 범위 경계
- 이번 단계에서 하지 않을 것:
  - 카카오톡 투표/캘린더 API 자동 연동
  - 다중 관리자 체계
  - 실시간 채팅
- 미래 확장:
  - 카카오 공유 문구 자동 생성
  - 초대/승인 알림 자동화
  - 운영 대시보드 지표
