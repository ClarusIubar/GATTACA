# 03. 사용자 연구와 문제 이해

## 사용자 집단
- 주요 사용자:
  - 카카오톡 단체방에서 함께 일정을 정하는 소규모 모임 구성원
- 우선 범위:
  - 이미 서로를 아는 관계, 소규모 커뮤니티, 추억 공유 니즈가 있는 그룹
- 제외 사용자:
  - 공개 커뮤니티 전체 대상 서비스
  - 익명 참여가 필요한 대규모 사용자

## 문제 상황
- 단체방에서 일정이 결정돼도 이후 사진과 후기가 채팅방 위로 흩어진다.
- 과거 모임을 다시 찾아보려면 메시지 검색, 사진첩, 메모를 여러 곳에서 확인해야 한다.
- 무엇을 했고 왜 좋았는지에 대한 맥락이 사라진다.

## 기존 대안의 한계
- 카카오톡 채팅과 사진첩은 기록 단위가 이벤트 중심으로 정리되지 않는다.
- 캘린더는 일정 관리에는 좋지만 추억 기록에는 과하다.
- 노션/문서는 공동 기록에 쓸 수 있지만 접근성과 감성 표현이 다소 무겁다.

## 연구 방법
- 운영자 경험 기반 문제 정의
- 유사 서비스 사용 패턴 관찰
- 카카오톡 그룹 운영 현실 분석
- AI 보조를 통한 사용자 흐름 정리

## 핵심 인사이트
- 사용자는 새 협업도구보다 이미 쓰는 단체방 흐름을 유지하길 원한다.
- 기록은 일정 관리보다 행사 후 회상 경험에 더 큰 가치를 가진다.
- 강한 권한 제어보다 “우리만 쓰는 공간”이라는 신뢰 모델이 중요하다.
