# 04. 제품 요구사항과 백로그

## 제품 목표

추억열차 v1은 카카오톡 단체방에서 이미 결정된 약속을 오래 보관할 수 있는 메모리얼 페이지로 구현한다. 캘린더 후보 조율 도구가 아니라, 확정된 일정을 정거장으로 등록하고 그날의 사진과 코멘트를 연결하는 기록소가 목표다.

## 핵심 기능

- 이벤트 등록: 언제, 어디서, 무엇을, 어떻게 할지와 확정 요약을 기록한다.
- 이벤트 목록/상세 조회: 월간 캘린더가 아니라 정거장형 타임라인으로 보여준다.
- 메모리 등록: 이벤트에 사진 URL 또는 업로드 이미지와 캡션, 기록 시각을 연결한다.
- 코멘트 등록/수정: 메모리 하위에 후기를 남긴다.
- 운영자 승인/반려: 카카오 로그인 사용자를 pending에서 approved/rejected로 관리한다.
- 운영자 삭제: 이벤트, 메모리, 코멘트 삭제는 운영자만 수행한다.
- Kakao relay: 브라우저가 직접 Kakao API를 호출하지 않고 Worker relay를 사용한다.

## 보조 기능

- demo 권한 전환 모드
- 이벤트, 메모리, 코멘트 수정
- Cloudflare Pages SPA 라우팅
- runtime-status 기반 배포 상태 표시
- Kakao OAuth secret 미설정 시 안전한 안내 화면

## v1 제외 기능

- 카카오톡 단체방 멤버 자동 판별
- 카카오톡 투표/캘린더 API 자동 연동
- 즉시 알림 보장
- 다중 관리자 역할
- 참여자별 행동 통계

## 비기능 요구사항

- 보안: 승인 사용자만 CRU, 운영자만 delete.
- 배포: `main` 머지 코드가 production으로 배포되어야 한다.
- UX: 모바일과 데스크톱에서 기본 CTA, 폼, 상태 메시지가 읽혀야 한다.
- 문서화: PRD, UI/UX, 아키텍처, 테스트, 배포, SDD 추적 문서가 실제 구현과 일치해야 한다.
- 확장성: Kakao relay 경계와 Worker 구조를 유지해 이후 자동화 확장이 가능해야 한다.
